Ruby's Different Variables

I touched upon this in my last technical post, but let's go more in depth with this topic. After all, in an object-oriented language like Ruby, variables are the objects you're working with most.

So, Ruby has four variable types - local, instance, class, and global variables. Let's take a closer look a each of these:

Local variable
A local variable is just that, local. That means it is only that specific variable within the code block it is defined. For example, if the local variable is assigned in a method, it isn't available outside that method. A local variable has to start with either an underscore or a lower case letter.
Here's what a local variable might look like -

j_sal = 1

Global variable
The good thing about variables in Ruby, is they're pretty self-explanatory. A local variable is local, so you'd imagine a global variable would be...yep, global! A global variable has a global scope, meaning it can be accessed anywhere in the entire Ruby program. Global variables are always prefixed by this sigil - "$" aka the dollar sign. Apparently it can get messy if you use these too much, though, because, yes, the global variables can be accessed anywhere in the program, but they can also be changed from anywhere in the program.
Here's what a global variable might look like -

$welcome = "Welcome brethren!"

Instance variable
The instance variable (very helpful in classes) is a variable preceded by @ - e.g. @variable - that can be used across methods within a particular object (or class!).
Here's what a class using instance variables may look like:

Class Mammal
 def initialize (hair, live_birth)
  @hair = hair
  @birth = live_birth
 end
end

Class variableClass variables are a bit tricky. Like the instance variable, a class variable is preceded by @'s, but they have two instead of one. E.g. @@class_variable
They're also like the global variable, though, in that they have a wider scope than the other two, essentially allowing you to use the variable across all like objects of the same class. In lay terms, this means you can use the class variable in a situation where an object/method may change the value of the class variable, and you would then have that new value for the class variable if it is used elsewhere.
